# Design Principles

Extracted from `src/app/globals.css`, `src/components/`, and rendered UI (S01–S09).

| # | Principle | Evidence | Implementation rule | Do | Don't |
|---|---|---|---|---|---|
| 1 | **Warm-neutral foundation** | `--bg: #FAFAF7`; `--ink: #1A1A17`; all neutral grays carry warm undertone (warm-gray palette with brownish cast) | Every background and text color must come from the warm-neutral token family. Do not introduce cool-gray neutrals or pure black/white without mapping them to a system token. | Use `--md-sys-color-background` (#FAFAF7) as the page canvas; use `--md-sys-color-surface` (#FFFFFF) to lift cards. | Replace warm-grays with pure grays (`#f5f5f5`, `#333`) or introduce cool blue-tinted neutrals not in the palette. |
| 2 | **Operational density** | `--density-row: 36px`; `--density-pad: 16px`; compact mode at `28px`/`10px`; 14px base font; table rows `10px 14px`; kanban uses packed `8px` card gap | This is a dense operational tool, not a marketing dashboard. All components default to compact-ish proportions. Use density tokens; never inflate padding to make UI "feel premium". | Stick to `--density-row` for control heights; use `14px` body text; keep table row padding tight (8–10px). | Inflate component heights to 44px+ or add spacious 24px internal padding for aesthetic reasons. |
| 3 | **Monochrome primary with violet accent** | `--accent: oklch(0.52 0.14 282)` violet; used only on active states, CTAs, avatars, progress; all other surfaces neutral | Accent appears only on interactive affordances (active nav icon, primary button, avatar bg, selected calendar day, drag-over tint). Not used as a section background or decorative tint. | Apply accent to `.btn-primary`, `.sb-item[data-on]` icon, `.sb-avatar`, `.cal-day.selected`. | Use accent as a card background, section highlight, or general decorative tint. |
| 4 | **Semantic status colors** | `--st-todo/progress/review/done/block` — 5 distinct oklch status colors used consistently across kanban dots, hour-overrun, due-date overdue, delta indicators | Each status color maps to exactly one semantic state. Use the status system token, not the raw oklch value, in component implementations. | Apply `--md-sys-color-status-error` for overdue/overrun; `--md-sys-color-status-success` for done/delta-up. | Reuse status colors as decorative or categorical colors, or invent a new semantic color without a status token. |
| 5 | **Categorical dept/member color coding** | `hue-c1..c8`; 8 oklch hues at uniform chroma/lightness; mapped to DEPT_HUE and member `hue` fields; used as `color-mix()` tinted backgrounds + foregrounds | Department and member colors always come from the categorical hue family. The 8 hues are enumerated — do not introduce new hue slots. Use the `hue()` utility or component token to derive tinted bg from the hue foreground. | Derive badge bg as `color-mix(in oklab, var(--md-ref-color-cat-N) 14%, transparent)`. | Add a 9th categorical hue or use hardcoded hex colors for dept/member identification. |
| 6 | **Surface hierarchy without gradients** | `.bg` → `.bg-elev` → `.surface` → `.surface-2` creates 4-level hierarchy purely through lightness steps; no gradients on surfaces; no glass/blur effects (except drawer scrim `backdrop-filter: blur(2px)`); elevation via `shadow-sm/md/lg` | Depth is communicated via surface lightness + shadow, not gradient. The only blur is the overlay scrim behind modals/drawers, not on content surfaces. | Use `box-shadow: var(--md-sys-elevation-sm)` on floating cards; use `var(--md-sys-color-surface)` vs `var(--md-sys-color-background-elevated)` to separate stacking layers. | Add gradient fills, glassmorphism, or blurred content surfaces. The drawer scrim blur is the sole exception. |
| 7 | **CJK-first typography** | `Noto Sans TC` with PingFang TC fallback; `lang="zh-Hant"`; `font-feature-settings: "ss01","cv11"`; tabular-nums on all data; tight tracking on display numerics (`-0.02em`) | All text uses the CJK-capable Noto Sans TC stack. All numeric/financial data must use tabular-nums. Use weight 500–600 for UI labels; weight 400 for body prose. Tighten tracking only on display-size numerics (22px+). | Set `font-feature-settings: "ss01","cv11"` on `<body>`; apply `font-variant-numeric: tabular-nums` on `.tnum`/any numeric data class. | Use a Latin-only typeface as fallback, omit tabular-nums on numeric columns, or apply negative tracking to 12–14px labels. |
