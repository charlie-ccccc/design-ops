DESIGN_PRINCIPLES.md

設計原則

萃取自 src/app/globals.css、src/components/ 及已渲染 UI（S01–S09）。

#原則依據實作規則應做不應做
1暖中性基礎--bg: #FAFAF7; --ink: #1A1A17；所有中性灰色帶有暖色調（帶棕色調的暖灰色調色板）每個背景色和文字顏色必須來自暖中性 token 系列。請勿引入冷灰中性色或純黑/白，除非對應至 system token。使用 --md-sys-color-background（#FAFAF7）作為頁面畫布；使用 --md-sys-color-surface（#FFFFFF）提升卡片。以純灰色（#f5f5f5、#333）取代暖灰色，或引入調色板中沒有的冷藍色調中性色。
2操作密度--density-row: 36px; --density-pad: 16px; compact mode at 28px/10px; 14px base font; table rows 10px 14px; kanban uses packed 8px card gap這是密集操作工具，而非行銷儀表板。所有元件預設為緊湊比例。使用密度 token；絕不為了讓 UI「感覺高端」而增大內距。控制項高度遵循 --density-row；使用 14px 內文文字；保持表格行內距緊湊（8–10px）。以審美理由將元件高度增至 44px+ 或新增寬鬆的 24px 內部內距。
3單色主體搭配紫色強調--accent: oklch(0.52 0.14 282) 紫色；僅用於啟用狀態、CTA、頭像、進度；其他所有表面為中性色強調色僅出現在互動元件上（啟用的導航圖示、主要按鈕、頭像背景、選中的日曆日、拖曳覆蓋淡色）。不用作區段背景或裝飾淡色。將強調色套用至 .btn-primary、.sb-item[data-on] 圖示、.sb-avatar、.cal-day.selected。將強調色用作卡片背景、區段高亮或通用裝飾淡色。
4語義狀態顏色--st-todo/progress/review/done/block——5 種不同的 oklch 狀態顏色，統一用於看板點、工時超支、截止日逾期、差異指標每個狀態顏色恰好對應一個語義狀態。在元件實作中使用 status system token，而非原始 oklch 值。對逾期/超支使用 --md-sys-color-status-error；對完成/差異上升使用 --md-sys-color-status-success。將狀態顏色重新用作裝飾或分類顏色，或在沒有 status token 的情況下發明新語義顏色。

前綴

層級前綴職責
Reference--md-ref-*僅原始數值——無語義或元件名稱
System--md-sys-*共用語義角色——僅參照 ref token

繼承關係

component token -> system token -> reference value

component token 不得跳層直接指向 ref token。system token 名稱中不得包含元件的特定結構名稱。

Reference 色階

100 (lightest) -> 0 (darkest)
  • 100 為最淺可見階（白色或近白色）。
  • 0 為最深可見階（近黑色）。
  • 暖中性色調：階 100、97、95、93、75、58、42、18、10。
  • 冷暗色調（暗色模式）：階 93、79、54、37、23、14、11、9、6。
  • 強調色（紫色）：步階 95、74、66、52。
  • 分類色：非明度漸進——以色相命名（cat-1..8）。
  • 狀態顏色：非明度漸進——以語義角色命名。

暗色模式在 [data-theme="dark"] 下覆蓋相同的 system token 名稱。冷暗色調的 reference token 單獨定義，並透過暗色覆蓋區塊以別名形式對應至 system token。

密度模式

PAGE_COMPOSITION_RULES.md

頁面組成規則

依據來源：src/app/globals.css（S01）、src/app/page.tsx（S01）、src/app/layout.tsx（S02）。

視窗與應用外殼

  • 主要視窗：桌面版 ≥ 768px；固定應用程式外殼，含側邊欄 + 主要內容
  • 應用程式外殼格線：grid-template-columns: 232px 1fr; height: 100vh
  • 區域：
  • .sidebar — 232px，固定寬度，全視窗高度，可滾動溢出
  • .main — 流體，min-width 0，flex 欄：.topbar（固定 56px 最小高度）+ .body（flex 1，可滾動）
  • .body — 可滾動的主要內容區域；背景 --md-sys-color-background-elevated（#F4F3EE）
  • 行動版（≤768px）：側邊欄隱藏在螢幕外，點擊漢堡選單時從上方滑入；覆蓋遮罩；主區域佔滿全寬
  • body 上的 overflow: hidden——刻意設計；捲動由各區域自行管理，而非頁面層級

版面節奏

  • 頁面邊距：水平 22px，垂直 18px（所有主要視圖：看板、儀表板、歷史）
  • 區段間距（儀表板格線）：14px
  • 卡片堆疊間距（看板）：欄內卡片間 8px
  • 頂部欄高度：最小 56px；內距 14px 22px

所有主要視圖均以 padding: 18px 22px 開始——非密度層級原因請勿變更此設定。

區段規則

  • 包含多行結構化資料（表格、圖表、請假清單）的區段以 .panel 包裹（白色表面、12px 圓角、1px 框線）
  • 以畫布層級為主的區段（看板、儀表板格線）不以面板包裹——它們直接置於提升的頁面背景上
  • 格線中面板間距使用 gap: 14px；看板欄內卡片間距使用 gap: 8px

字型排版組合規則

  • KPI 指標鎖定：標籤 → 數值 → 差異指標；垂直堆疊；標籤始終全大寫；同一卡片中的標籤-數值-差異指標層級不得拆散
  • 面板標頭：font: 600 14px；可選搭配 12px 淡色副標題；兩者位於水平 flex 列中，右側工具有間隔
  • 頂部欄標題：600 16px；可選在下方加上 12px 淡色麵包屑
  • 抽屜標題：18px/600；上方附有 12px/500 id 徽章
  • 表格標頭：始終使用 uppercase; letter-spacing: 0.06em; font-weight: 600; 12px; color: muted
  • 顯示數字（22–28px）：始終使用 tabular-nums; letter-spacing: -0.02em——大尺寸下緊縮字距
  • 數值旁的資料標籤：始終小於數值（12px 標籤 → 18–28px 數值）；絕不相同大小

導航區域

側邊欄

  • 寬度：232px；背景：--md-sys-color-background（與頁面背景相同的暖白色，非純白）
  • 右側框線：1px solid --md-sys-color-divider
  • 結構：品牌區域 → 群組標籤 + 項目 → 間隔 → 使用者頭像
  • 群組標籤（.sb-group-h）：12px/600/大寫/+0.08em 字距；muted-2 顏色
  • 導航項目（.sb-item）：14px/500；圖示 24px；間距 10px；圓角 9px；懸停背景 surface-variant；啟用：surface 背景 + elevation-sm + primary 圖示顏色
  • 底部使用者區塊：頭像（28px 色相圓形）+ 姓名 + 角色；始終固定在底部

頂部導航欄

  • 背景：--md-sys-color-background（非 surface——與側邊欄背景視覺融合）
  • 底部框線：1px solid --md-sys-color-divider
  • 左側：漢堡選單（僅行動版）→ 標題 + 麵包屑
  • 右側：工具列（搜尋欄或圖示按鈕 + 操作按鈕 + 通知）
  • 桌面版：所有工具列項目可見；行動版：收合為圖示按鈕，搜尋在頂部欄下方的次要篩選列

卡片與表面

  • 應做：對資料表格、圖表容器和請假清單使用面板（.panel）
  • 不應做：每個畫面區段都以面板包裹；看板和儀表板使用直接格線版面
  • 不應做：在有框線的面板內嵌套有框線的面板（最多一層表面提升）
  • 看板卡：暖色提升底（#F4F3EE）上的白色表面——對比在預設狀態下無需額外陰影即可產生視覺區隔
  • 卡片懸停：elevation-sm + 較強的框線（而非改變卡片主體顏色）

清單與行

  • 表格行：padding: 8–10px 14px；border-bottom: 1px solid var(--md-sys-color-divider)
  • 請假行：padding: 8px 16px；懸停背景 → surface-variant
  • 每行不使用獨立的卡片/面板處理——行是輕量的，非方框式
  • 寬表格（cap-table, xtab）首欄固定：position: sticky; left: 0; z-index: 2
  • 表格頁尾：font-weight: 600; bg: surface-variant; border-top: 1px solid outline-strong

捲動與固定區域行為

  • 側邊欄：獨立可滾動（overflow-y: auto）——長導航清單獨立滾動
  • 看板：在 body 層級水平滾動（min-width: max-content）——欄位不換行
  • 儀表板：在 .body 內垂直滾動
  • 交叉分析表：最大高度 460px，內部滾動（overflow: auto）
  • 抽屜本體：overflow-y: auto；頁首 + 頁尾固定在抽屜內
  • 應用程式 body：overflow: hidden——無文件層級滾動

ANTI_AI_STYLE_RULES.md

Anti-AI 風格規則

CMoneyDesign 設計部工作看板的專案特定限制。在產生任何 UI 建議前先套用。

非有依據請勿新增

  • 任何內容表面的背景漸層（側邊欄品牌標誌使用了微妙的漸層；那是唯一的例外）
  • 內容表面的毛玻璃效果（唯一的模糊效果是模態/抽屜遮罩）
  • 光暈、球形、散景、SVG 抽象波紋、動態背景
  • 含行銷文案的通用 SaaS 主視覺區段
  • 裝飾性插圖面板
  • 嵌套有框線卡片——最多一層表面
  • 行即卡片處理——資料行是行，非卡片
  • 超出觀測密度的額外空白（這是操作工具；密度是功能的一部分）
  • 超出已定義的 4 個圓角步階（6、9、12、16px）的額外步階
  • 超出已定義的 3 個陰影層級（sm、md、lg）的額外層級
  • 過度飽和的強調色淡色表面（紫色強調色僅出現在啟用狀態、頭像和 CTA 上）

色彩限制

  • 未有新部門實體對應時，請勿引入第 9 種分類顏色
  • 請勿將強調色用於區段背景淡色（例如紫色橫幅、淡色側邊欄）
  • 請勿假設自訂背景色使用白色前景——僅在紫色（--md-sys-color-primary）和確認的深色表面上使用 --md-sys-color-on-primary（白色）
  • 請勿對以下情況以外使用 status-error：阻擋/超出工時、逾期截止日、負差異指標、阻擋的欄狀態
  • 請勿在 5 狀態系統以外新增語義顏色（琥珀警告、資訊藍等），除非新產品狀態需要
  • 暗色模式表面中性色比亮色模式更偏冷色（略帶藍色調）；請勿對暗色模式表面使用暖中性色十六進位值

字型排版限制

版面規則

  • 容器內距：16px 18px（非對稱：上下 16px，左右 18px；使用 ref 尺寸）
  • Flex 方向：欄；標籤、數值、差異指標間 gap: 4px
  • .kpi-val（數值插槽）margin-top: 6px——標籤後的額外空間
  • 數值和單位在同一視覺行內為 inline-flex；單位在 margin-left: 4px
  • 差異指標行：flex 列，align-items: center，gap: 4px
  • 卡片沒有固定高度——高度由內容決定

內容規則

  • label：zh-Hant 或英文均全大寫；最多 16 個字元；必須傳達指標類別而非數值（如 本月工時、完成率、任務數）
  • value：整數或最多 1 位小數；始終使用 font-variant-numeric: tabular-nums；目前不使用千位分隔符
  • unit：可選後綴字串：H（工時）、%、件——數值和單位間無空格
  • delta：簡短描述，例如 ↑ +12%、較上月 −3H；保持在 20 字元以下

無障礙規則

  • 語義角色：純顯示的統計面板使用 <div>，無需 ARIA role
  • 螢幕閱讀器：數值必須依序可讀——標籤 + 數值 + 單位；請勿以裝飾性元素分隔
  • 數字：font-variant-numeric: tabular-nums 僅為視覺效果；確保底層文字為可讀的數字字元
  • 差異指標圖示（箭頭/角形）：若使用，標記 aria-hidden="true"；僅以文字顏色傳達方向是不足的——新增描述方向的視覺隱藏文字
